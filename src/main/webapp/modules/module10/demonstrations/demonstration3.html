Démonstration 3 L'attribution d'un nom de domaine

Configuration du nom de domaine

Dans le cadre du cours la configuration du nom de domaine est réalisé dans le fichier hosts (C:\Windows\System32\drivers\etc) comme ceci:

Paramétrage du serveur Tomcat

Voici la configuration à réaliser dans le fichier server.xml pour que l'application soit accessible au travers de l'URL www.javaee.eni.fr

Observez les 3 modifications apportées:

  1. La modification de l'attribut name de la balise <Host> avec le nom de domaine voulu.
  2. La modification de l'attribut path de la balise <Constext> pour que l'application soit accessible directement avec le nom de domaine.
  3. La modification de l'attribut defaultHost de la balise <Engine> afin d'avoir un comportement cohérent en de requête émise avec une adresse ip au lien d'un nom de domaine.

	 	
<?xml version="1.0" encoding="UTF-8"?>
<Server port="-1" shutdown="SHUTDOWN">
  <Listener className="org.apache.catalina.startup.VersionLoggerListener" />
  <Listener className="org.apache.catalina.core.AprLifecycleListener" SSLEngine="on" />
  <Listener className="org.apache.catalina.core.JreMemoryLeakPreventionListener" />
  <Listener className="org.apache.catalina.mbeans.GlobalResourcesLifecycleListener" />
  <Listener className="org.apache.catalina.core.ThreadLocalLeakPreventionListener" />

	<Service name="Catalina">

		<Connector port="80" protocol="HTTP/1.1"/>

		<Engine name="Catalina" defaultHost="www.javaee.eni.fr">

			<Host 	name="www.javaee.eni.fr"  
					appBase="applis_localhost"
					unpackWARs="true" 
					autoDeploy="false"
					deployOnStartup="false">

				<Context docBase="AppliDemo.war" path="">
			
				</Context>
			</Host>
		</Engine>
	</Service>
</Server>
