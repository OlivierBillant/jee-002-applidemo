Démonstration 2 Le déploiement explicite

Organisation du système de fichiers

Tomcat peut éventuellement héberger plusieurs hôtes. Chacun de ces hôtes hébergeant plusieurs applications. Il est donc judicieux de créer un répertoire au nom de chaque hôte pour y déposer les applications à déployer.

Supprimez le répertoire webapps contenant les applications disponibles par défaut avec Tomcat. Elle ne sont pas nécessaires en production.

Créez ensuite un répertoire pour l'hôte localhost. Vous pouvez le nommer applis_localhost. Déposez l'archive de l'application à déployer dans ce répertoire :

Configuration dans le fichier server.xml :

Voici la configuration minimaliste à réaliser dans le fichier server.xml pour que l'application soit accessible au travers de l'URL http://localhost/Tuto 

	 	
<?xml version="1.0" encoding="UTF-8"?>
<Server port="-1" shutdown="SHUTDOWN">
  <Listener className="org.apache.catalina.startup.VersionLoggerListener" />
  <Listener className="org.apache.catalina.core.AprLifecycleListener" SSLEngine="on" />
  <Listener className="org.apache.catalina.core.JreMemoryLeakPreventionListener" />
  <Listener className="org.apache.catalina.mbeans.GlobalResourcesLifecycleListener" />
  <Listener className="org.apache.catalina.core.ThreadLocalLeakPreventionListener" />

	<Service name="Catalina">

		<Connector port="80" protocol="HTTP/1.1"/>

		<Engine name="Catalina" defaultHost="localhost">

			<Host 	name="localhost"  
					appBase="applis_localhost"
					unpackWARs="true" 
					autoDeploy="false"
					deployOnStartup="false">

				<Context docBase="AppliDemo.war" path="/Tuto">
			
				</Context>
			</Host>
		</Engine>
	</Service>
</Server>
